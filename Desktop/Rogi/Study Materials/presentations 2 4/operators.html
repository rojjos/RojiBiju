Java Operators

At the lowest level, data in Java is manipulated using operators.

Using Java operators

An operator takes one or more arguments and produces a new value. Addition and unary plus (+), subtraction and unary minus (-), multiplication (*), division (/),and assignment (=)

Using Java operators

All operators produce a value from their operands. In addition, some operators change the value of an operand. This is called a side effect. 


int x = 5; // initialize to 5
x = 23;    // assign (copy or store) the value 23 in x

Using Java operators

Almost all operators work only with primitives. Exceptions:‘=‘, ‘==‘ and ‘!=‘, which work with all objects and the String class supports ‘+’ and ‘+=‘

Precedence

Operator precedence defines how an expression evaluates when several operators are present follow the order of operations. Use () to show precedence

Assignment

Assignment is performed with the operator =, but you cannot assign anything to a constant value 


a = 4;

//no good
4 = a;

Assignment

Since the primitive holds the actual value and not a reference to an object, when you assign primitives, you copy the contents from one place to another

Assignment

Aliasing

Aliasing - fundamental way that Java works with objects. Whenever you manipulate an object, what you’re manipulating is the reference, so when you assign “from one object to another,” you’re actually copying a reference from one place to another.

Assignment

Aliasing during method calls

Aliasing will also occur when you pass an object into a method

Mathematical operators

The basic mathematical operators are the same as the ones available in most programming languages: addition (+) subtraction (-) division (/) multiplication (*) modulus (%, which produces the remainder from integer division). Integer division truncates, rather than rounds, the result.

Mathematical operators

Java also uses the shorthand notation from C/C++ that performs an operation and an assignment at the same time. 


x += 4

Mathematical operators

Unary minus and plus operators

The unary minus (-) and unary plus (+) are the same operators as binary minus and plus. 


//Straight forward
x = -a;

//The compiler is able to figure out
x = a * -b;

//clearer to say
x = a * (-b);
Unary minus inverts the sign on the data. Unary plus provides symmetry with unary minus, although it doesn’t have any effect.

Auto increment and decrement

Shortcuts can make code much easier to type, and either easier or harder to read. The decrement operator is -- The increment operator is ++

There are two versions of each type of operator, often called the prefix and postfix versions. Preincrement means the ++ operator appears before the variable, post-increment means the ++ operator appears after the variable

Relational operators

Relational operators generate a boolean result.
The relational operators are:

  • less than (<)
  • greater than (>)
  • less than or equal to (<=)
  • greater than or equal to (>=)
  • equivalent (==) and not equivalent (!=)

Relational operators

Testing object equivalence

The operators == and != compare object references. Use the special method equals( ) to compare the actual contents of an object. The default behavior of equals( ) is to compare references.

Logical operators

Each of the logical operators AND (&&), OR (||) and NOT (!) produces a boolean value of true or false based on the logical relationship of its arguments.

Logical operators

Short-circuiting

This means that the expression will be evaluated only until the truth or falsehood of the entire expression can be unambiguously determined.

Bitwise operators

The bitwise operators come from C’s low-level orientation, where you often manipulate hardware directly and must set the bits in hardware registers. Java was originally designed to be embedded in TV set-top boxes, so this low-level orientation still made sense. However, you probably won’t use the bitwise operators much.

Shift operators

The shift operators also manipulate bits. They can be used solely with primitive, integral types.

Ternary if-else operator

The ternary operator, also called the conditional operator, is unusual because it has three operands.


boolean-exp ? value0 : value1

String operator + and +=

The + and += operators can be used to concatenate strings

Common pitfalls when using operators

One of the pitfalls when using operators is attempting to leave out the parentheses when you are even the least bit uncertain about how an expression will evaluate. 


while(x = y) {
// ....
}

Casting operators

The word cast is used in the sense of “casting into a mold.” Java will automatically change one type of data into another when appropriate.

Casting operators

Truncation and rounding

Casting a value always truncates the number. If instead you want the result to be rounded, use the round( ) methods

Casting operators

Promotion

If you perform any mathematical or bitwise operations on primitive data types that are smaller than an int (that is, char, byte, or short), those values will be promoted to int before performing the operations, and the resulting value will be of type int.

Casting operators

Promotion

In general, the largest data type in an expression is the one that determines the size of the result of that expression; if you multiply a float and a double, the result will be double; if you add an int and a long, the result will be long.
